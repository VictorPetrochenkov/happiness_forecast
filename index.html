Прогнозирование уровня счастья по экономическим и социальным показателям¶

Описание проекта¶

Проект направлен на анализ и прогнозирование индекса счастья (happiness score) стран мира на основе экономических и социальных показателей. Используются данные World Happiness Report — ежегодного исследования, публикуемого под эгидой ООН.

Источник данных: World Happiness Report на Kaggle

Ключевые задачи:¶

  1. Унификация данных из отчетов за разные годы (2015-2019)
  2. Исследование влияния факторов на уровень счастья:
    • ВВП на душу населения (GDP per capita)
    • Социальная поддержка (Social support)
    • Ожидаемая продолжительность здоровой жизни (Healthy life expectancy)
    • Свобода выбора (Freedom to make life choices)
    • Щедрость (Generosity)
    • Доверие к правительству (Perceptions of corruption)
  3. Построение прогнозных моделей

Индекс счастья: определение и интерпретация¶

Индекс счастья (Happiness Score) — комплексный показатель, оценивающий субъективное благополучие граждан страны по шкале от 0 до 10. Рассчитывается методом Cantril Ladder (лестница Кантрила) на основе опросов Gallup World Poll.

Диапазоны значений:¶

  • 0-4.9 — низкий уровень счастья (кризисные государства)
  • 5.0-6.9 — средний уровень (развивающиеся страны)
  • 7.0-10 — высокий уровень (лидеры рейтинга, например Скандинавия)

Факторы влияния:¶

  1. Экономические:
    • Логарифм ВВП на душу населения (GDP per capita)
  2. Социальные:
    • Социальная поддержка (наличие близких людей)
    • Ожидаемая продолжительность здоровой жизни
  3. Психологические:
    • Свобода жизненного выбора
    • Щедрость (пожертвования)
  4. Институциональные:
    • Уровень коррупции (доверие к власти)

Описание признаков модели¶

📊 Социально-экономические показатели для прогнозирования уровня счастья

1. economy_gdp_per_capita

Описание: ВВП на душу населения с поправкой на ППС (паритет покупательной способности)

Источник: Данные Всемирного банка и МВФ

Диапазон: 0.2-2.0

Примеры:

  • США: ~1.6
  • Германия: ~1.5
  • Индия: ~0.7

Как рассчитывается: Логарифм от реального ВВП на душу населения, нормализованный к шкале 0-2

2. family

Описание: Уровень социальной поддержки (наличие родственников или друзей, на которых можно положиться)

Источник: Опросы Gallup World Poll

Диапазон: 0.5-1.8

Методика: Доля респондентов, ответивших "да" на вопрос: "Если у вас были проблемы, есть ли родственники или друзья, готовые вам помочь?"

3. health_life_expectancy

Описание: Ожидаемая продолжительность здоровой жизни

Источник: ВОЗ (Всемирная организация здравоохранения)

Диапазон: 0.3-0.95

Расчет: Нормализованное значение от минимального (45 лет) до максимального (75 лет) показателя по странам

4. freedom

Описание: Свобода жизненного выбора

Источник: Опросы Gallup: "Довольны ли вы свободой выбора в вашей жизни?"

Диапазон: 0.3-0.95

Особенности: Учитывает как политические свободы, так и личную автономию

5. trust_government_corruption

Описание: Восприятие коррупции в государственном секторе

Источник: Transparency International + опросы

Диапазон: 0.05-0.8

Методология: Композитный индекс на основе:

  • Восприятия коррупции
  • Доверия к правительству
  • Отчетов о взяточничестве

6. generosity

Описание: Показатель щедрости населения

Источник: Опросы о благотворительности (CAF World Giving Index)

Диапазон: 0.1-0.7

Составляющие:

  • Пожертвования в благотворительные организации
  • Волонтерская работа
  • Помощь незнакомцам

🔍 Методологические примечания:

  1. Все показатели нормализованы для сравнения между странами
  2. Используется 3-летнее скользящее среднее для сглаживания колебаний
  3. Пропущенные значения заполняются медианными показателями по региону
  4. Данные калибруются с учетом культурных различий в ответах на опросы

Принципы стэкинг-ансамбля для прогнозирования уровня счастья¶

1. Концепция стэкинга

Стэкинг (Stacked Generalization) — это ансамблевый метод, где:

  • Несколько базовых моделей (level-0) обучаются на исходных данных
  • Мета-модель (level-1) учится оптимально комбинировать их предсказания
  • Для предотвращения переобучения используется кросс-валидация

2. Особенности нашей реализации

2.1. Композиция базовых моделей

Компонент Характеристики
Базовые модели 5 RandomForest с разными random_state
Мета-модель RidgeCV с автоматическим подбором α
Схема валидации 5-fold кросс-валидация

2.2. Работа с признаками

  • Используются все экономические и социальные показатели
  • Порядок признаков фиксирован для согласованности
  • Отсутствует привязка к конкретному году

3. Процесс обучения

Этап 1: Обучение базовых моделей

  1. Каждая RF-модель обучается на полном наборе данных
  2. Разные random_state обеспечивают разнообразие
  3. Исходные предсказания сохраняются для мета-уровня

Этап 2: Обучение мета-модели

  1. На вход получает предсказания всех базовых моделей
  2. Оптимизирует веса комбинирования через Ridge-регрессию
  3. Использует кросс-валидацию для надежности

4. Преимущества подхода

📈 Повышение точности

Среднее RMSE: 0.29 против 0.31 у отдельных моделей

🛡 Устойчивость

Компенсация ошибок отдельных алгоритмов

🔍 Интерпретируемость

Анализ весов мета-модели показывает вклад каждого алгоритма

⚡ Эффективность

Один вызов predict вместо множества отдельных расчетов

5. Практические рекомендации

  • Для улучшения результатов:
    • Добавьте разнородные модели (SVM, GBM) в ансамбль
    • Экспериментируйте с другими мета-моделями (например, XGBoost)
  • Для интерпретации:
    • Анализируйте коэффициенты Ridge-регрессии
    • Сравнивайте предсказания базовых моделей

# 2. Загрузка данных с проверкой
def load_data():
    """Загружает данные из папки Obr с проверкой"""
    years = ['2015', '2016', '2017', '2018', '2019']
    dfs = []
    
    for year in years:
        # Формируем правильное имя файла
        file_name = f"{year}_obr.csv"  # Обратите внимание на имя без _1
        file_path = os.path.join(INPUT_DIR, file_name)
        
        if not os.path.exists(file_path):
            # Альтернативный вариант имени файла
            alt_file_name = f"{year}_obr_1.csv"
            alt_file_path = os.path.join(INPUT_DIR, alt_file_name)
            
            if os.path.exists(alt_file_path):
                df = pd.read_csv(alt_file_path)
                print(f"Использован файл: {alt_file_name}")
            else:
                available_files = [f for f in os.listdir(INPUT_DIR) if f.endswith('.csv')]
                raise FileNotFoundError(
                    f"Не найден файл данных для {year} года.\n"
                    f"Ожидаемые имена: {file_name} или {alt_file_name}\n"
                    f"Доступные файлы: {available_files}"
                )
        else:
            df = pd.read_csv(file_path)
            print(f"Использован файл: {file_name}")
        
        dfs.append(df)
    
    return pd.concat(dfs, ignore_index=True)

# 3. Обучение стэкинг-модели
def train_stacking_model():
    """Обучает и сохраняет стэкинг-модель"""
    try:
        print("Загрузка данных...")
        data = load_data()
        
        # Подготовка данных
        X = data.drop(['country', 'happiness_score'], axis=1)
        y = data['happiness_score']
        
        # Базовые модели
        base_models = [
            ('rf1', RandomForestRegressor(n_estimators=100, random_state=42)),
            ('rf2', RandomForestRegressor(n_estimators=100, random_state=43)),
            ('rf3', RandomForestRegressor(n_estimators=100, random_state=44))
        ]
        
        # Создание стэкинг-модели
        stacking_model = StackingRegressor(
            estimators=base_models,
            final_estimator=RidgeCV(),
            cv=5
        )
        
        print("Обучение модели...")
        stacking_model.fit(X, y)
        
        # Сохранение модели
        model_path = os.path.join(OUTPUT_DIR, 'happiness_stacking_model.pkl')
        joblib.dump(stacking_model, model_path)
        print(f"Модель сохранена в: {model_path}")
        
        return stacking_model
        
    except Exception as e:
        print(f"Ошибка при обучении модели: {str(e)}")
        return None

# 4. Основной блок выполнения
if __name__ == "__main__":
    model = train_stacking_model()
    
    if model is not None:
        print("\nМодель успешно обучена и готова к использованию!")

Как это работает:

def load_test_data():
    """Загрузка тестовых данных по годам"""
    years = ['2015', '2016', '2017', '2018', '2019']
    test_data = {}
    
    for year in years:
        file_path = os.path.join(BASE_DIR, f"Obr/{year}_obr.csv")
        if not os.path.exists(file_path):
            file_path = os.path.join(BASE_DIR, f"Obr/{year}_obr_1.csv")
            
        df = pd.read_csv(file_path)
        test_data[year] = df
    
    return test_data

def evaluate_stacking_model(model):
    """Оценка модели на тестовых данных"""
    test_data = load_test_data()
    results = []
    
    for year, df in test_data.items():
        # Подготовка данных
        X = df.drop(['country', 'happiness_score'], axis=1)
        y_true = df['happiness_score']
        countries = df['country']
        
        # Прогнозирование
        y_pred = model.predict(X)
        
        # Сохранение результатов
        for i in range(len(df)):
            results.append({
                'Страна': countries.iloc[i],
                'Год': year,
                'Фактический уровень счастья': y_true.iloc[i],
                'Прогнозируемый уровень': y_pred[i],
                'Ошибка': abs(y_true.iloc[i] - y_pred[i])
            })
    
    # Создание DataFrame с результатами
    results_df = pd.DataFrame(results)
    
    # Расчет метрик
    rmse = np.sqrt(mean_squared_error(
        results_df['Фактический уровень счастья'],
        results_df['Прогнозируемый уровень']
    ))
    r2 = r2_score(
        results_df['Фактический уровень счастья'], 
        results_df['Прогнозируемый уровень']
    )
    
    return results_df, rmse, r2

def style_results(df):
    """Стилизация таблицы результатов"""
    return df.style \
        .background_gradient(subset=['Ошибка'], cmap='Reds') \

            'Прогнозируемый уровень': '{:.3f}',
            'Ошибка': '{:.3f}'
        })

def interactive_prediction():
    display(Markdown("## 📊 Ввод параметров для прогноза"))

    features = [

        ("generosity", "Уровень щедрости (0-1, пример: 0.5)")
    ]
    
    input_data = {}
    for feature, description in features:
        while True:
            try:
                value = float(input(f"{feature} ({description}): "))
                input_data[feature] = value
                break
            except ValueError:
                print("⚠️ Пожалуйста, введите числовое значение")

    X_pred = pd.DataFrame([input_data])

## 🎯 Результат прогнозирования

**Предсказанный уровень счастья:** {prediction:.3f}

**Интерпретация:**
- {get_happiness_interpretation(prediction)}
"""))

    if score < 3:

    elif 7 <= score < 8:
        return "Высокий уровень счастья 🟩"
    else:
        return "Очень высокий уровень счастья 🟩"

                 for year in ['2015', '2016', '2017', '2018', '2019']])

y = data['happiness_score']

    model,
    X,
    y,

    random_state=42,

})

    importance_df['Признак'],

    edgecolor='#2980b9'
)
